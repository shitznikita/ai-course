# День 3: Разные способы рассуждения

Минимальный Kotlin CLI, который решает одну логическую задачу через LLM API четырьмя способами и печатает сравнение.

Задача:

```text
Есть 12 монет, одна из них фальшивая и легче остальных.
Как найти фальшивую монету за 3 взвешивания на чашечных весах?
```

Почему эта задача подходит: правильность решения можно проверить. Нужно не просто дать общий совет, а описать алгоритм, который гарантированно находит одну легкую монету среди 12 за 3 взвешивания.

## Что используется

- Язык: Kotlin
- Интерфейс: CLI
- API: Eliza API
- Модель: `gpt-5-mini`
- HTTP-клиент: `java.net.http.HttpClient`
- Параметры генерации: стандартные для модели
- Ключ: только через `.env` или переменные окружения

`temperature` в коде не задается, потому что `gpt-5-mini` через Eliza вернул ошибку: поддерживается только значение по умолчанию. Это нормальный пример того, что параметры LLM API отличаются у разных моделей и провайдеров.

## План

1. Взять одну проверяемую логическую задачу.
2. Отправить ее в модель напрямую.
3. Отправить ту же задачу с инструкцией "решай пошагово".
4. Попросить модель сначала составить промпт для решения задачи.
5. Использовать сгенерированный промпт для решения исходной задачи.
6. Отправить ту же задачу группе экспертов: аналитик, инженер, критик.
7. Сделать отдельный comparison-запрос к модели-судье.
8. Вывести все ответы в консоль.

## Чек-лист

- [ ] Одна и та же задача используется во всех четырех подходах.
- [ ] Есть режим `DIRECT`.
- [ ] Есть режим `STEP BY STEP`.
- [ ] Есть режим `PROMPT FIRST`.
- [ ] Есть режим `EXPERTS`.
- [ ] Есть секция `COMPARISON`.
- [ ] Используется LLM API через REST.
- [ ] API-ключ не хранится в коде.
- [ ] Используется одна и та же модель во всех режимах.

## Настройка API-ключа

Можно использовать `.env` из дня 1. Скрипт `run-eliza.sh` сам подхватит `day-01-llm-rest-kotlin/.env`, если в папке дня 3 нет своего `.env`.

Если нужен отдельный `.env` для дня 3:

```bash
cp day-03-reasoning-methods-kotlin/.env.example day-03-reasoning-methods-kotlin/.env
```

И вставь токен:

```dotenv
LLM_API_KEY=your_eliza_oauth_token_here
LLM_AUTH_SCHEME=OAuth
LLM_API_URL=https://api.eliza.yandex.net/raw/openai/v1/chat/completions
LLM_MODEL=gpt-5-mini
```

Файл `.env` нельзя коммитить.

## Запуск

Команды запускаются из корня репозитория.

Если truststore для Eliza еще не создан:

```bash
day-03-reasoning-methods-kotlin/scripts/setup-yandex-ca.sh
```

Запуск:

```bash
day-03-reasoning-methods-kotlin/scripts/run-eliza.sh
```

Можно передать свою задачу:

```bash
day-03-reasoning-methods-kotlin/scripts/run-eliza.sh --args="Твоя логическая задача"
```

Сборка:

```bash
./gradlew :day-03-reasoning-methods-kotlin:build
```

## Сценарий видео

1. Открыть `Main.kt` и показать, что код делает обычные REST-запросы через `HttpClient`.
2. Показать, что API-ключ берется из `.env`, а не хранится в коде.
3. Показать одну общую задачу про 12 монет.
4. Показать четыре режима: direct, step-by-step, prompt-first, experts.
5. Запустить:

```bash
day-03-reasoning-methods-kotlin/scripts/run-eliza.sh
```

6. Показать в консоли четыре ответа и секцию `COMPARISON`.
7. Коротко сказать, какой подход дал самый проверяемый ответ.

## Проверка требований

- Одна задача во всех подходах: да.

- Есть direct: да.
- Есть step-by-step: да.
- Есть prompt-first: да.
- Есть experts: да.
- Есть comparison: да.
