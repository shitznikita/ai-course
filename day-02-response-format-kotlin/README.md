# День 2: Формат ответа

Минимальный Kotlin CLI, который отправляет один и тот же пользовательский запрос к LLM API в пяти вариантах:

1. `WITHOUT LIMITS` — обычный запрос без явного контроля ответа.
2. `FORMAT ONLY` — добавлен только явный формат ответа.
3. `LENGTH ONLY` — добавлено только ограничение длины.
4. `FINISH ONLY` — добавлено только условие завершения.
5. `ALL LIMITS` — формат, длина и завершение одновременно.

Запрос отправляется обычным REST `POST` через `java.net.http.HttpClient`. Готовые AI-агенты и высокоуровневые LLM SDK не используются.

## Что используется

- Язык: Kotlin
- Интерфейс: CLI
- API: Eliza API
- Модель: `gpt-5-mini`
- HTTP-клиент: `java.net.http.HttpClient`
- Ключ: только через `.env` или переменные окружения

## План

1. Взять один основной пользовательский prompt.
2. Отправить его в модель без дополнительных ограничений.
3. Отправить тот же prompt с одним ограничением на формат: JSON.
4. Отправить тот же prompt с одним ограничением на длину: не больше 80 слов.
5. Отправить тот же prompt с одним условием завершения: последняя строка `END`.
6. Отправить тот же prompt со всеми ограничениями сразу.
7. Вывести пять ответов в консоль рядом.
8. Сравнить, как каждое ограничение меняет поведение модели.

## Почему такой вариант

Для `gpt-5-mini` через Eliza проще и надежнее управлять ответом через prompt-инструкции, потому что не все параметры одинаково поддерживаются разными моделями и провайдерами. В дне 1 уже была ошибка с `max_tokens`, поэтому в этом решении контроль сделан через system/user prompt:

- `system prompt` задает общее поведение;
- `user prompt` содержит тот же основной запрос и конкретные ограничения;
- завершение ответа реализовано явной инструкцией: строка `END` или поле `"finish": "END"`.

Это соответствует заданию: контроль можно делать через параметры API или через prompt.

## Чек-лист

- [ ] Один и тот же основной prompt используется во всех пяти режимах.
- [ ] Есть режим `WITHOUT LIMITS`.
- [ ] Есть режим `FORMAT ONLY`.
- [ ] Есть режим `LENGTH ONLY`.
- [ ] Есть режим `FINISH ONLY`.
- [ ] Есть режим `ALL LIMITS`.
- [ ] Формат JSON показан отдельно и вместе с другими ограничениями.
- [ ] Лимит 80 слов показан отдельно и вместе с другими ограничениями.
- [ ] Завершение `END` показано отдельно и вместе с другими ограничениями.
- [ ] Все пять ответов выводятся в консоль.
- [ ] API-ключ не хранится в коде.

## Настройка API-ключа

Можно использовать `.env` из дня 1. Скрипт `run-eliza.sh` сам подхватит `day-01-llm-rest-kotlin/.env`, если в папке дня 2 нет своего `.env`.

Если нужен отдельный `.env` для дня 2:

```bash
cp day-02-response-format-kotlin/.env.example day-02-response-format-kotlin/.env
```

И вставь токен:

```dotenv
LLM_API_KEY=your_eliza_oauth_token_here
LLM_AUTH_SCHEME=OAuth
LLM_API_URL=https://api.eliza.yandex.net/raw/openai/v1/chat/completions
LLM_MODEL=gpt-5-mini
```

Файл `.env` нельзя коммитить.

## Запуск

Команды запускаются из корня репозитория.

Если truststore для Eliza еще не создан:

```bash
day-02-response-format-kotlin/scripts/setup-yandex-ca.sh
```

Запуск:

```bash
day-02-response-format-kotlin/scripts/run-eliza.sh
```

Можно передать свой prompt:

```bash
day-02-response-format-kotlin/scripts/run-eliza.sh --args="Объясни, зачем API возвращать структурированный JSON"
```

Сборка:

```bash
./gradlew :day-02-response-format-kotlin:build
```

## Пример ожидаемого вывода

```text
Model: gpt-5-mini
Prompt: Объясни начинающему разработчику, зачем управлять форматом ответа LLM через API.

=== WITHOUT LIMITS ===
<свободный ответ модели>

=== FORMAT ONLY ===
{
  "summary": "...",
  "details": ["...", "...", "..."]
}

=== LENGTH ONLY ===
<короткий ответ не больше 80 слов>

=== FINISH ONLY ===
<свободный ответ модели>
END

=== ALL LIMITS ===
{
  "summary": "...",
  "rules": ["...", "...", "..."],
  "finish": "END"
}
```

## Сценарий видео

1. Открыть `Main.kt` и показать список из пяти режимов запроса.
2. Показать, что основной prompt один и тот же, а меняются только дополнительные ограничения.
3. Показать ограничения: JSON-формат, лимит 80 слов, завершение `END`.
4. Запустить:

```bash
day-02-response-format-kotlin/scripts/run-eliza.sh
```

5. Показать в консоли блоки `WITHOUT LIMITS`, `FORMAT ONLY`, `LENGTH ONLY`, `FINISH ONLY`, `ALL LIMITS`.
6. Коротко сказать: "Это один и тот же запрос к LLM API. Я отдельно показываю влияние формата, длины и условия завершения, а затем применяю все ограничения сразу".

## Проверка требований

- REST-запрос к LLM API: да.
- Один и тот же основной prompt во всех пяти режимах: да.
- Ответ без ограничений: да.
- Ответы с отдельными ограничениями: да.
- Ответ со всеми ограничениями: да.
- Явный формат ответа: JSON.
- Ограничение длины: инструкция "не больше 80 слов".
- Условие завершения: `END` / `"finish": "END"`.
- Сравнение ответов в консоли: да.
- API-ключ не хардкодится: да.
